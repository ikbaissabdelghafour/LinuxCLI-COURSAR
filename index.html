شرح هيكلية أوامر لينكس (Linux Command Structure)
الصيغة العامة:
command [OPTIONS] [ARGUMENTS]
  • command: اسم الأمر المراد تنفيذه (مثل ls, cat).
  • [OPTIONS]: خيارات تبدأ غالباً بشرطة (- أو --) لتغيير سلوك الأمر (مثال: -l).
  • [ARGUMENTS]: ملفات أو مسارات أو قيم إضافية يحتاجها الأمر (مثال: file.txt).
مثال: ls -l /home/user
  • ls: الأمر
  • -l: خيار (عرض بشكل مفصل)
  • /home/user: مسار المجلد
ملاحظات:
  • يمكن دمج الخيارات القصيرة: ls -la
  • بعض الأوامر تحتاج صلاحيات root (استخدم sudo)
  • استخدم man command لعرض دليل المساعدة لأي أمر
ما هو لينكس؟ (What is Linux?)
لينكس هو نظام تشغيل حر ومفتوح المصدر، يُستخدم في الخوادم، الحواسيب الشخصية، الأجهزة الذكية، وأكثر.
يتميز بالاستقرار، الأمان، والمرونة العالية، ويعتمد بشكل أساسي على سطر الأوامر لإدارة النظام.
توزيعات لينكس متنوعة مثل: Ubuntu, Debian, Fedora, Arch، وغيرها.
ما هو النواة (Kernel)?
النواة (Kernel) هي القلب ديال نظام التشغيل لينكس.
كتلعب دور الوسيط بين الهاردوير (المعالج، الذاكرة، الأجهزة) والسوفتوير (البرامج).
أهم المهام ديالها:
  • إدارة الذاكرة (Memory Management)
  • إدارة العمليات (Process Management)
  • التعامل مع الأجهزة (Device Drivers)
  • تأمين النظام (Security)
شجرة لينكس (Linux File System Tree)
ف لينكس كلشي منظم فـ / (الجذر). هادو بعض المجلدات المهمة:
  • / : الجذر (Root) — البداية ديال جميع الملفات
  • /bin : أوامر أساسية (ls, cp, mv, cat...)
  • /boot : ملفات الإقلاع (Bootloader, Kernel)
  • /etc : ملفات الإعدادات والتكوينات
  • /home : مجلدات المستخدمين
  • /lib : مكتبات ضرورية للنظام والبرامج
  • /media و/mnt : لتركيب الأقراص الخارجية
  • /opt : برامج إضافية
  • /root : مجلد المستخدم root
  • /sbin : أوامر إدارية (لـ root)
  • /tmp : ملفات مؤقتة
  • /usr : برامج ومكتبات مستخدمين
  • /var : سجلات (Logs) وملفات متغيرة
  • /dev : تمثيل الأجهزة (مثل الأقراص الصلبة)
  • /proc : معلومات عن العمليات والنواة
أساسيات يجب معرفتها في لينكس
  • كل شيء في لينكس عبارة عن ملف (ملفات، مجلدات، أجهزة...)
  • الملفات والمجلدات منظمة في شجرة جذرية (/)
  • الصلاحيات (قراءة، كتابة، تنفيذ) تُدار عبر أوامر مثل chmod وchown
  • المستخدم العادي لا يملك صلاحيات كاملة، استخدم sudo لتنفيذ أوامر إدارية
  • الأوامر تُنفذ في الطرفية (Terminal)
  • يمكنك استخدام tab للإكمال التلقائي للأوامر والمسارات
  • استخدم man وinfo للحصول على المساعدة
  • الملفات المخفية تبدأ بنقطة (.)
  • يمكنك إعادة توجيه المخرجات باستخدام > و>> و|
الرموز في سطر الأوامر (CLI Symbols)
الرمز الاسم / النوع الوظيفة مثال
| Pipe يمرر المخرجات ديال أمر كمدخل لأمر آخر ls | grep txt
&& AND ينفذ الأمر الثاني غير إذا نجح الأول mkdir test && cd test
|| OR ينفذ الأمر الثاني غير إذا فشل الأول cd folder || echo "ما كاينش"
& Background يشغل الأمر فـ الخلفية ping google.com &
; Separator ينفذ عدة أوامر بالتتابع cd /; ls; pwd
> Redirect (Write) يكتب المخرجات فـ ملف (يمسح القديم) ls > files.txt
>> Redirect (Append) يزيد المخرجات لآخر الملف echo "test" >> log.txt
2> Redirect Errors يحفظ الأخطاء فـ ملف ls notfound 2> errors.txt
* Wildcard يعني أي عدد من الحروف ls *.txt
? Wildcard يعني أي حرف واحد ls file?.txt
~ Home يشير للمجلد الشخصي cd ~
. Current Directory المجلد الحالي ./script.sh
.. Parent Directory المجلد الأعلى cd ..
